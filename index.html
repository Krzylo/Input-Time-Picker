jQuery TimePicker pligin for browsers that do not support the input type="time" field

Add code in head tag
< script type="text/javascript" src="https://ajax.googleapis.com/ajax/libs/jquery/3.1.1/jquery.min.js" ></script>
< script type="text/javascript" src="jquery.timepiker.js"></script>
< link href="style.min.css" rel="stylesheet" type="text/css" />
< script >
$(document).ready(function(){
$(":input[type=text]").timepicker({ showPicker: true, // show buttons : reset, up, down ( true / false )
disableMenu: true, // enable or disable contextmenu ( true / false )
autoComplete: "off"}); // autoComplite ( off / on )
});
< /script >

For input type="time" we need to check if the browser supports this field, for example use modernizr.js script.
In head tag add line:
< head >
. . .
<script type="text/javascript" src="modernizr.js"></script>
. . .
< /head >
Check it
< script >
$(document).ready(function(){
if ( ! Modernizr.inputtypes.time ) {
// not-supported
$(":input[type=time]").timepicker();
} });
< /script >

Enabled contextmenu
< script >
$(":input[type=text]").timepicker({showPicker: true, disableMenu: false, autoComplete: "off"});
< /script >

Disabled buttons, Time is entering from the numeric keypad
< script >
$(":input[type=text]").timepicker({showPicker: false, disableMenu: true, autoComplete: "off"});
< /script >
